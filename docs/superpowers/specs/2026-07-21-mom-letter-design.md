# 엄마에게 보내는 스크롤 편지 페이지 — 디자인

## 목적

엄마에게 보낼 감동적인 편지를, 스크롤에 따라 봉투가 열리고 문장이 하나씩 드러나는 인터랙티브 웹 페이지로 만든다. 완성 후 링크로 전송해 엄마가 폰/PC 브라우저에서 볼 수 있게 한다.

## 스토리 구조 (5 씬)

스크롤 진행률(0~1, 전체 페이지 스크롤 길이 기준)을 마스터 값으로 삼아 3D 오브젝트와 텍스트가 같은 값을 참조해 항상 싱크되도록 한다.

| # | 장면 | 3D 오브젝트 동작 | 텍스트 |
|---|------|------------------|--------|
| 1 | 봉투 (시작) | 닫힌 봉투가 화면 중앙에서 살짝 공중부양하며 천천히 회전 | 없음. 스크롤 유도 화살표만 표시 |
| 2 | 봉투 열림 | 스크롤 진행에 따라 봉투 뚜껑이 힌지 회전으로 열리고, 안에서 편지지가 슬라이드 아웃 | 없음 |
| 3 | 수신인 | 봉투가 fade out, 카메라가 pull back, 꽃/하트 오브젝트가 주변에 등장하기 시작 | "엄마에게" 페이드인 |
| 4 | 편지 본문 | 꽃/하트가 천천히 회전·부유하며, 문장이 넘어갈 때마다 은은하게 강조(스케일업/색 변화) | 문장을 스크롤 구간별로 하나씩 배정. 해당 구간에서만 페이드인 → 페이드아웃 후 다음 문장으로 교체 |
| 5 | 마무리 서명 | 꽃/하트가 서서히 한곳으로 모이거나 잦아듦 | 서명 문구("OO 올림" 등) 페이드인, 화면에 고정 |

문장 개수는 실제 편지 내용이 확정되면 맞춘다. 지금은 placeholder 문장으로 구조를 채운다.

## 기술 스택

- **Three.js** (CDN, 빌드 스텝 없음) — 봉투, 꽃, 하트 3D 오브젝트 렌더링
- **GSAP + ScrollTrigger** (CDN) — 스크롤 위치를 하나의 타임라인으로 만들어 3D 오브젝트 트랜스폼과 DOM 텍스트 페이드를 같은 값으로 구동
- 3D 오브젝트는 외부 모델(glTF 등) 없이 Three.js geometry로 코드에서 직접 생성 (low-poly/스타일라이즈드 형태). 다운로드 없이 바로 로딩, 로딩 속도 가장 빠름
- 파일 구성: `index.html`, `style.css`, `scene.js`(Three.js 셋업/오브젝트 정의), `app.js`(스크롤 타임라인, 텍스트 등장 로직)

## 데이터 흐름

1. GSAP ScrollTrigger가 페이지 전체 스크롤 진행률(0~1)을 추적
2. 진행률 값을 씬 구간별(1~5)로 매핑하는 헬퍼 함수가 현재 씬과 씬 내부 진행률을 계산
3. `scene.js`의 update 함수가 이 값을 받아 카메라 위치, 봉투 힌지 각도, 꽃/하트 회전·위치를 갱신
4. `app.js`가 같은 값을 받아 현재 표시할 문장의 DOM 요소 opacity/transform을 갱신 (이전 문장은 페이드아웃, 다음 문장은 페이드인)
5. 두 로직 모두 같은 스크롤 진행률을 소스로 삼기 때문에 3D와 텍스트가 어긋나지 않음

## 에러 처리 / 폴백

- WebGL 미지원 브라우저 감지 시 정적 CSS 애니메이션(또는 이미지) 버전으로 폴백
- 3D 에셋이 코드 생성 방식이라 로딩 실패 케이스 자체가 거의 없음 (외부 파일 의존 없음)

## 성능 (모바일 우선)

엄마가 폰으로 볼 가능성이 높으므로:
- low-poly 지오메트리 유지
- `renderer.setPixelRatio`를 `Math.min(window.devicePixelRatio, 2)` 등으로 캡
- 꽃/하트 등 반복 오브젝트 개수를 구형 기기에서도 부드럽게 돌아가는 선에서 제한
