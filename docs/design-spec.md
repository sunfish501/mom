# 디자인 스펙 — 엄마 생신 축하 편지 페이지 (Deluxe Edition)

> 대상: 개발자 (다음 단계). 이 문서대로 정적 사이트(HTML/CSS/JS + CDN)를 구현합니다.
> 뷰어: 어머니 한진영 님 (주로 휴대폰으로 열람). 작성자: 아들 정의겸 (디미고 재학생).
> 목표: 엄마가 열어보고 "와, 역시 우리 아들" 할 만큼 **화려하면서도 깔끔하고 예쁜** 한 장의 편지.

---

## 0. 한눈에 보기 (개발자용 요약)

- **정적 사이트**: HTML + CSS + Vanilla JS. **빌드 스텝 없음.** CDN 라이브러리는 적극 사용 OK. GitHub Pages 배포.
- **컨셉**: 밤하늘/별빛 위에 떠오르는 손편지. 그라디언트 + 반짝이는 파티클 배경 + 글래스모피즘 카드 + 스크롤 트리거 애니메이션 + 서명부 컨페티 피날레.
- **사진 없음**: 타이포그래피 + 빛/모션으로만 감동 전달. 사진 슬롯 만들지 않음.
- **모바일 우선**: 엄마 폰이 최우선. 화려한 이펙트가 **가독성·성능을 절대 해치면 안 됨**.
- **편지 본문은 비워둠**: 정의겸이 직접 씀. 7번 플레이스홀더 마커를 **그대로** 넣고, 이 영역은 어떤 이펙트로도 가려지지 않게 함.
- **핵심 원칙 "화려하지만 깔끔"**: 배경/장식은 화려하게, **편지 텍스트 자체는 깨끗하고 정적으로**. 대비로 고급스러움을 만든다.

### 추천 CDN 스택 (전부 정적, 빌드 불필요)
| 용도 | 라이브러리 | 비고 |
|---|---|---|
| 스크롤 트리거 등장 | **AOS** (Animate On Scroll) | 가장 가볍고 선언적. `data-aos` 속성만. 1순위 |
| (대안/고급) 스크롤·타임라인 | GSAP + ScrollTrigger | 더 정교한 연출 필요 시. 과용 금지 |
| 반짝이는 파티클 배경 | **tsParticles** (slim 빌드) | 별/반짝임. 모바일에선 파티클 수 축소 |
| 서명부 피날레 | **canvas-confetti** | 마지막 등장 시 1회 발사 |
| 웹폰트 | Google Fonts | Nanum Myeongjo 등 |

> 라이브러리는 **필요한 것만**. AOS + canvas-confetti + (선택) tsParticles 조합만으로 목표 달성 가능. 로딩 부담되면 tsParticles 대신 순수 CSS 별빛(아래 3-1 대안)으로 대체.

## 1. 전체 컨셉 / 무드

**"깊은 밤하늘, 별이 조용히 반짝이는 가운데 빛으로 쓴 손편지가 한 장씩 떠오른다."**

- 배경: 딥 네이비 → 자수정/플럼으로 흐르는 은은한 그라디언트, 그 위에 작은 별빛이 천천히 반짝임(과하지 않게).
- 편지 내용은 **반투명 유리(글래스모피즘) 카드** 위에 또렷하게. 배경은 화려해도 글자는 항상 선명.
- 스크롤을 내리면 각 문장이 아래에서 부드럽게 떠오르고, 강조 단어엔 은은한 골드 글로우.
- 마지막 서명("아들 정의겸 올림")이 나타나는 순간 **컨페티가 한 번 터지며** 축하의 정점을 찍는다.
- 정서 키워드: 벅참, 반짝임, 고급스러움, 진심, 그리고 깔끔함. 촌스럽지 않게, 세련되게.

## 2. 페이지 구조 / 섹션 순서 (골격 유지)

세로 단일 페이지. 배경 레이어(그라디언트+파티클)는 `position: fixed`로 전 구간 공통, 콘텐츠가 그 위를 스크롤.

| 순서 | 섹션 | 내용 | 애니메이션 |
|---|---|---|---|
| 0 | **Background Layer** (전역) | 그라디언트 + 별빛 파티클 | 항상 은은히 움직임(자동, 저강도) |
| 1 | **Intro (인사)** | "엄마," 타이핑됨. 하단 스크롤 힌트(↓). | **타자기 효과** + 커서 깜빡임, 등장 시 카드 살짝 떠오름 |
| 2 | **Birthday (생신 축하)** | "생신 진심으로 축하드려요" (고정 텍스트 가능). 숫자/키워드 골드 글로우. | 스크롤 진입 fade+slide-up, 강조어 글로우 펄스 |
| 3 | **Letter Body (편지 본문 — 플레이스홀더)** | **정의겸이 직접 쓸 자리.** 여러 문단. 글래스 카드 위, 깨끗한 정적 텍스트. | 문단별 순차 등장(stagger). 텍스트 자체엔 요란한 이펙트 금지 |
| 4 | **Sign-off (맺음말) + 피날레** | "사랑을 담아, 아들 정의겸 올림" | 조용히 페이드인 → **canvas-confetti 1회 발사** |

**흐름 원칙**
- Intro 타이핑은 자동 재생. 이후 섹션은 스크롤 트리거(AOS/IntersectionObserver), 한 번 등장하면 유지.
- 컨페티는 서명 섹션이 처음 뷰포트에 들어올 때 **딱 1회**(재발사 금지 — `observer.unobserve`).
- 화려한 연출은 배경/전환에, 편지 본문 텍스트는 항상 정적·선명.

## 3. 애니메이션 / 비주얼 이펙트 상세

### 3-1. 배경 — 그라디언트 + 별빛
- 그라디언트: `linear-gradient(160deg, #0F0C29 0%, #2D1B4E 45%, #3B2063 100%)` (딥 네이비→플럼). 아주 느린 `background-position` 이동(`@keyframes`, 40s+)으로 살아있는 느낌.
- 별빛(택1):
  - **A. tsParticles(slim)**: 작은 흰/골드 점, 개수 데스크톱 60 / 모바일 25, 느린 opacity 트윙클, 상호작용(hover/click) OFF로 성능 보호.
  - **B. 순수 CSS 대안(경량)**: 여러 개의 작은 `radial-gradient` 점 + `@keyframes twinkle`로 opacity 깜빡임. 라이브러리 0개. 저사양 폰 안전.
- `prefers-reduced-motion: reduce` → 파티클/이동 정지, 정적 그라디언트만.

### 3-2. 글래스모피즘 카드
- 편지 콘텐츠 컨테이너: `background: rgba(255,255,255,0.08)`, `backdrop-filter: blur(14px)`, `border: 1px solid rgba(255,255,255,0.18)`, `border-radius: 20px`, 부드러운 그림자.
- `backdrop-filter` 미지원 브라우저 폴백: 불투명도 높인 단색 배경(`rgba(30,20,50,0.7)`)으로 가독성 확보.

### 3-3. 타자기(타이핑) 효과 — Intro
- Vanilla JS로 한 글자씩(40~70ms), 골드 커서 `|` 깜빡임. `prefers-reduced-motion`이면 전체 즉시 표시.

### 3-4. 스크롤 트리거 등장 — Birthday/Letter/Sign-off
- **AOS 권장**: 요소에 `data-aos="fade-up"`, `data-aos-delay`로 stagger. `once: true`로 1회만.
- AOS 미사용 시 IntersectionObserver + `.is-visible`(opacity/transform transition)로 동일 구현.
- 강조 단어: 골드 텍스트 + 아주 은은한 `text-shadow` 글로우 펄스(느리게, 1~2회 후 정지). 과하지 않게.

### 3-5. 서명부 피날레 — canvas-confetti
- 서명 섹션 최초 진입 시 `confetti()` 1회. 톤: 골드/크림/로즈 계열 색으로(브랜드 톤 맞춤), 좌우 2발 정도로 절제.
- 모바일 성능 위해 particleCount 적당히(150 이하), 1회 후 종료.

> **성능 가드레일**: 애니메이션은 `opacity`/`transform`/`filter` 위주(리플로우 회피). 라이브러리 총량 주시, 모바일에서 60fps 목표. 무거우면 파티클→CSS 대안, 컨페티 유지.

## 4. 색상 팔레트 & 타이포그래피

### 4-1. 색상 (딥 나이트 + 골드 포인트)
| 용도 | Hex | 설명 |
|---|---|---|
| 배경 그라디언트 시작 | `#0F0C29` | 딥 네이비/미드나잇 |
| 배경 그라디언트 중간 | `#2D1B4E` | 딥 퍼플 |
| 배경 그라디언트 끝 | `#3B2063` | 플럼/자수정 |
| 글래스 카드 | `rgba(255,255,255,0.08)` | 반투명 유리 |
| 본문 텍스트 | `#F5EFE6` | 따뜻한 아이보리 (딥 배경 위 고대비) |
| 강조/제목 | `#F3D27A` | 소프트 골드 |
| 포인트/글로우 | `#E9B949` | 진한 골드 (하트, 커서, 컨페티) |
| 서브/힌트 | `#B9A8D0` | 흐린 라벤더 (화살표, 날짜) |

- 본문 `#F5EFE6` on 딥 배경 = 충분한 대비(가독성 최우선). 편지 텍스트는 골드로 쓰지 말 것(강조어에만 골드).

### 4-2. 타이포그래피
- **한글 본문**: `Nanum Myeongjo` (명조, 편지다움) — Google Fonts CDN, `font-display: swap`.
  - 제목/서명 포인트에 한해 `Nanum Pen Script`(손글씨) 제한 사용 가능. 본문 전체엔 가독성 이유로 비권장.
  - 폴백: `'Nanum Myeongjo', 'Apple SD Gothic Neo', 'Malgun Gothic', serif`.
- **크기 (모바일 기준)**: 제목/Intro `2.0rem`·`lh 1.4` / 축하 `1.5rem` / 본문 `1.125rem`(≈18px)·`lh 1.9`(넉넉한 행간) / 서명 `1.25rem` / 서브 `0.9rem`.
- 자간 `0.02em`, 문단 여백 넉넉히, `word-break: keep-all`(한글 단어 단위 줄바꿈).

## 5. 반응형 / 모바일 우선

- 기본 스타일 = 세로 휴대폰. 미디어쿼리로 큰 화면만 보정.
- 콘텐츠/글래스 카드 폭 `max-width: 640px`, 좌우 `padding: 1.25rem`.
- `@media (min-width: 768px)`: 폰트 소폭 확대(제목 `2.5rem`, 본문 `1.2rem`), 파티클 수 증가 허용.
- 터치 친화(탭 요소 최소 44x44px). 이 페이지는 스크롤만으로 완성 — 필수 버튼 없음.
- 가로 스크롤 금지(`overflow-x: hidden`, 요소 `max-width: 100%`). `<meta viewport>` 필수.
- **모바일 성능 분기**: 화면폭/`navigator.hardwareConcurrency` 등으로 파티클 수·컨페티량 축소. `backdrop-filter` 부하 크면 blur 반경 축소.

## 6. 파일 구조 권장 (개발자용)

빌드 없이 GitHub Pages 루트에서 바로 서비스.

```
/ (repo root)
├── index.html        # 마크업 + 섹션. <head>에 Google Fonts & CDN(AOS/tsParticles/canvas-confetti) <link>/<script>
├── style.css         # 색상 변수(:root), 글래스모피즘, 그라디언트/별빛 키프레임, 반응형
├── script.js         # 타이핑 + AOS 초기화(또는 IntersectionObserver) + 파티클 설정 + 컨페티 트리거 + 모바일 성능 분기
├── content.js        # ★ 편지 텍스트만 모아둔 파일 (정의겸이 편집할 곳)
└── docs/
    └── design-spec.md
```

### content.js — 텍스트를 코드/이펙트에서 분리 (중요)
편지 문구를 레이아웃·애니메이션과 완전히 분리해, 정의겸이 **이 파일 하나만** 열어 수정하면 되도록.

```js
// content.js — 여기 있는 글자만 고치면 됩니다. HTML/CSS/이펙트는 건드릴 필요 없어요.
const LETTER = {
  intro: "엄마,",                      // 타이핑될 첫 인사
  birthday: "생신 진심으로 축하드려요.",   // 생신 축하 메시지 (수정 가능)

  // [편지 본문 자리 — 정의겸이 직접 작성 예정]
  // 문단을 배열로. 한 줄 = 한 문단. 원하는 만큼 추가/삭제하세요.
  body: [
    "[편지 본문 자리 — 정의겸이 직접 작성 예정]",
    "[여기에 두 번째 문단을 쓰세요 — 원하는 만큼 문단을 추가할 수 있어요.]"
  ],

  signoff: "사랑을 담아,\n아들 정의겸 올림"  // 맺음말 (고정)
};
```

- `script.js`는 `LETTER.body`를 순회해 `<p>`를 생성하고 스크롤 등장(stagger)을 붙임. **본문 텍스트 자체엔 골드/글로우/움직임 이펙트를 주지 않음** — 등장 페이드만.

## 7. 편지 본문 플레이스홀더 마커 (정확히 이대로 구현)

개발자는 편지 본문 자리를 **절대 임의로 채우지 말고**, 아래 마커를 그대로 넣어 정의겸이 쉽게 찾도록 함. **이 영역은 어떤 화려한 이펙트로도 가려지거나 읽기 어려워지면 안 됨.**

**본문 플레이스홀더 텍스트 (이 문자열 그대로):**
```
[편지 본문 자리 — 정의겸이 직접 작성 예정]
```

**HTML 안에 둘 경우 — 감싸는 주석도 함께:**
```html
<!-- ▼▼▼ 편지 본문 시작 — 정의겸이 직접 작성할 부분 ▼▼▼ -->
<!-- content.js 의 LETTER.body 배열을 수정하세요. 아래 문단은 예시 플레이스홀더입니다. -->
<p>[편지 본문 자리 — 정의겸이 직접 작성 예정]</p>
<!-- ▲▲▲ 편지 본문 끝 ▲▲▲ -->
```

- 개발 중 플레이스홀더는 화면에서 **눈에 띄게**(연한 배경/점선 테두리). 실제 텍스트가 채워지면 자연스러운 본문 스타일.
- 최종 편지 문구는 **정의겸 본인만** 작성. 개발자/테스터는 창작하지 않음.
- 가독성 체크: 글래스 카드 blur/대비가 본문을 흐리게 하지 않는지 반드시 확인.

## 9. 접근성 / 성능 / 마무리 체크

- `prefers-reduced-motion: reduce` 대응: 타이핑·슬라이드·파티클·컨페티 모두 축소/생략, 정적 표시.
- 색 대비 충분(아이보리 본문 vs 딥 배경). 강조 골드는 배경 대비 확인.
- 시맨틱 마크업(`<main>`, `<section>`, `<h1>`, `<p>`), `lang="ko"`.
- CDN 로드 실패해도 콘텐츠는 읽힘(이펙트 없이 graceful degradation): 폰트 폴백, AOS 없으면 즉시 표시, 파티클/컨페티 없으면 배경 그라디언트만.
- 모바일 실제 기기에서 60fps·가독성·로딩 속도 확인. 무거우면 파티클→CSS 대안으로 강등.
- 가로 스크롤 없음, 모든 화면폭에서 깨지지 않음.

> **한 줄 요약**: 배경은 밤하늘처럼 화려하게, 편지는 유리 위에 얹은 듯 깨끗하게. 서명에서 컨페티로 마침표. 그리고 무슨 일이 있어도 엄마 폰에서 잘 읽히고 잘 돌아가게.
