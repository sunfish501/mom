# 디자인 스펙 — 엄마 생신 축하 편지 페이지

> 대상: 개발자 (다음 단계). 이 문서대로 정적 사이트(HTML/CSS/JS)를 구현합니다.
> 뷰어: 어머니 한진영 님 (주로 휴대폰으로 열람). 작성자: 아들 정의겸.

---

## 0. 한눈에 보기 (개발자용 요약)

- **정적 사이트**: 순수 HTML + CSS + Vanilla JS. 빌드 스텝 없음. GitHub Pages 배포.
- **컨셉**: 3D/스크롤 봉투 없음. **타이핑(타자기) 애니메이션 + 부드러운 페이드/스크롤 등장** 중심의 조용하고 따뜻한 편지.
- **사진 없음**: 타이포그래피와 텍스트만으로 감정 전달. 사진 슬롯을 만들지 마세요.
- **모바일 우선**: 세로 화면 기준으로 먼저 설계, 데스크톱은 최대 폭만 제한.
- **편지 본문은 비워둠**: 정의겸이 나중에 직접 씁니다. 아래 7번의 플레이스홀더 마커를 **정확히 그대로** 넣으세요.

## 1. 전체 컨셉 / 무드

**"밤에 조용히 켜서, 한 줄 한 줄 천천히 읽는 손편지."**

- 화면이 열리면 배경은 따뜻한 크림/아이보리 톤. 화면 중앙에 글자가 **한 자 한 자 타이핑되듯** 나타난다.
- 요란한 전환이나 화려한 그래픽 없이, 종이에 만년필로 눌러쓴 듯한 정적이고 진심 어린 분위기.
- 스크롤을 내릴수록 인사 → 생신 축하 → 편지 본문 → 맺음말 순으로 문장이 **부드럽게 떠오른다(fade + 살짝 위로 이동)**.
- 색과 여백이 넉넉해서 눈이 편하고, 어머니가 천천히 음미하며 읽을 수 있는 호흡.
- 정서적 키워드: 따뜻함, 진심, 잔잔함, 감사, 편안함.

## 2. 페이지 구조 / 섹션 순서

세로로 이어지는 단일 페이지(single scroll). 각 섹션은 뷰포트에 들어올 때 등장 애니메이션.

| 순서 | 섹션 | 내용 | 애니메이션 | 타이밍 |
|---|---|---|---|---|
| 1 | **Intro (인사)** | "엄마," 한 단어 또는 짧은 인사가 타이핑됨. 첫인상. | **타자기 효과** (한 글자씩), 커서 깜빡임 | 페이지 로드 즉시 시작 |
| 2 | **Birthday (생신 축하)** | "생신 축하드려요" 등 축하 메시지. 이 부분은 고정 텍스트로 작성 가능. | 페이드인 + 위로 슬라이드. 강조 단어는 살짝 크게. | Intro 타이핑 완료 후 or 스크롤 진입 시 |
| 3 | **Letter Body (편지 본문 — 플레이스홀더)** | **정의겸이 직접 쓸 자리.** 여러 문단. | 문단별 스크롤 등장(fade/slide), 문단 사이 충분한 여백 | 스크롤로 각 문단이 뷰포트 진입 시 |
| 4 | **Sign-off (맺음말)** | "사랑을 담아, 아들 정의겸 올림" | 마지막에 조용히 페이드인 | 본문 마지막 문단 후 스크롤 진입 시 |

**흐름 원칙**
- 1번(Intro)은 자동 재생: 페이지를 열면 별도 조작 없이 타이핑이 시작된다.
- 2~4번은 스크롤에 반응해 등장(IntersectionObserver). 이미 지나간 섹션은 다시 사라지지 않음(한 번 등장하면 유지).
- 스크롤 유도가 필요하면 Intro 하단에 **아래 화살표(↓) 힌트**를 은은하게 표시(bounce 애니메이션), 스크롤 시작하면 사라짐.

## 3. 애니메이션 접근

CDN 라이브러리 없이 순수 CSS/JS로 충분히 구현 가능. (원하면 타이핑만 Typed.js 사용해도 되지만 **의존성 없이 직접 구현 권장** — 가볍고 GitHub Pages에서 안정적.)

### 3-1. 타자기(타이핑) 효과 — Intro
- Vanilla JS로 문자열을 인덱스 순회하며 `textContent`에 한 글자씩 추가. 글자 간 40~70ms.
- 텍스트 끝에 CSS로 깜빡이는 커서(`|`) 표현: `@keyframes blink`.
- 접근성/모션 민감: `prefers-reduced-motion: reduce`이면 타이핑 생략하고 전체 텍스트 즉시 표시.

### 3-2. 스크롤 등장(fade + slide-up) — Birthday / Letter / Sign-off
- 기본 상태: `opacity: 0; transform: translateY(24px);`
- 등장: `.is-visible` 클래스 부여 시 `opacity: 1; transform: translateY(0);` (`transition: 0.8s ease-out`).
- `IntersectionObserver`로 뷰포트 진입 감지(threshold ~0.15), 한 번 보이면 `unobserve`.
- 문단이 여러 개면 `transition-delay`로 순차 등장(stagger, 예: 0.1s씩).
- `prefers-reduced-motion: reduce`이면 transform 없이 즉시 표시.

### 3-3. 스크롤 힌트 화살표
- 순수 CSS `@keyframes`로 위아래 bounce. 스크롤 시작(`scrollY > 0`) 감지되면 페이드아웃.

> 성능: 이미지/폰트 최소화, 애니메이션은 `opacity`/`transform`만 사용(리플로우 없음). 모바일에서 부드럽게.

## 4. 색상 팔레트 & 타이포그래피

### 4-1. 색상 (따뜻한 크림/세피아 계열)

| 용도 | Hex | 설명 |
|---|---|---|
| 배경 (Background) | `#FDF6EC` | 따뜻한 크림/아이보리, 종이 느낌 |
| 배경 보조/카드 | `#F7EBDA` | 살짝 진한 크림 (섹션 구분 필요 시) |
| 본문 텍스트 | `#4A3F35` | 부드러운 다크 세피아 (순검정 대신) |
| 강조 텍스트/제목 | `#8C5A3C` | 따뜻한 브라운/테라코타 |
| 포인트 (하트/장식) | `#C96B5B` | 은은한 로즈/테라코타 레드 |
| 서브/힌트 텍스트 | `#A8998A` | 흐린 세피아 (커서, 화살표, 날짜 등) |

- 다크 배경은 사용하지 않음(따뜻한 라이트 톤 고정). 어머니 세대에게 밝고 편안한 화면.
- 대비: 본문 `#4A3F35` on `#FDF6EC`는 충분한 명도 대비 확보(가독성 우선).

### 4-2. 타이포그래피

- **한글 본문 폰트**: 손편지/따뜻함을 위해 **부드러운 명조 또는 손글씨풍**.
  - 1순위: `Nanum Myeongjo` (명조, 정중하고 편지다움) — Google Fonts CDN.
  - 손글씨 느낌을 원하면 `Nanum Pen Script`를 **제목/맺음말 포인트**로만 제한 사용(본문 전체엔 가독성 이유로 비권장).
  - 폴백: `'Nanum Myeongjo', 'Apple SD Gothic Neo', 'Malgun Gothic', serif`.
- **크기 (모바일 기준, rem)**
  - Intro/제목: `2.0rem` (약 32px), `line-height: 1.4`
  - 생신 축하 메시지: `1.5rem`
  - 본문: `1.125rem`(약 18px), `line-height: 1.9` — 어머니가 읽기 편하게 **넉넉한 행간**.
  - 맺음말: `1.25rem`, 살짝 강조.
  - 서브/힌트: `0.9rem`
- 자간(letter-spacing) 살짝(`0.02em`) 주어 편지 느낌. 문단 간 여백 넉넉히(`margin`).
- Google Fonts는 `<link>` CDN으로 로드(빌드 불필요). 로딩 지연 대비 `font-display: swap`.

## 5. 반응형 / 모바일 우선

- **모바일 우선 설계**: 기본 스타일 = 세로 휴대폰. 미디어쿼리로 큰 화면만 보정.
- **콘텐츠 폭**: 본문 컨테이너 `max-width: 640px`, 좌우 `padding: 1.25rem`(모바일에서 화면에 붙지 않게).
- **브레이크포인트**
  - 기본(모바일): 위 4-2 크기.
  - `@media (min-width: 768px)`(태블릿/데스크톱): 폰트 살짝 키우고(제목 `2.5rem`, 본문 `1.2rem`), 컨테이너 중앙 정렬 유지, 좌우 여백 확대.
- **터치 친화**: 클릭/탭이 필요한 요소는 최소 44x44px. (이 페이지는 상호작용이 거의 없어 스크롤만으로 완성되게 — 필수 버튼 없음.)
- 세로 스크롤만 사용, 가로 스크롤 절대 없음(`overflow-x: hidden`, 모든 요소 `max-width: 100%`).
- `<meta name="viewport" content="width=device-width, initial-scale=1">` 필수.
- 긴 문장 줄바꿈 자연스럽게(`word-break: keep-all`로 한글 단어 단위 줄바꿈 — 어색한 중간 끊김 방지).

## 6. 파일 구조 권장 (개발자용)

빌드 없이 GitHub Pages 루트에서 바로 서비스되도록 **저장소 루트에 배치**.

```
/ (repo root)
├── index.html        # 마크업 + 섹션 구조. <head>에 Google Fonts <link>.
├── style.css         # 전체 스타일, 색상 변수(:root CSS custom properties), 반응형, 애니메이션 키프레임
├── script.js         # 타이핑 효과 + IntersectionObserver 스크롤 등장 + 스크롤 힌트
├── content.js        # ★ 편지 텍스트 데이터만 모아둔 파일 (정의겸이 편집할 곳)
└── docs/
    └── design-spec.md
```

### content.js — 텍스트를 코드에서 분리 (중요)
편지 문구를 레이아웃/로직과 분리해, 정의겸이 **이 파일 하나만** 열어 수정하면 되도록 합니다.

```js
// content.js — 여기 있는 글자만 고치면 됩니다. HTML/CSS는 건드릴 필요 없어요.
const LETTER = {
  intro: "엄마,",                    // 타이핑될 첫 인사
  birthday: "생신 진심으로 축하드려요.", // 생신 축하 메시지 (수정 가능)

  // [편지 본문 자리 — 정의겸이 직접 작성 예정]
  // 문단을 배열로. 한 줄 = 한 문단. 원하는 만큼 추가/삭제하세요.
  body: [
    "[편지 본문 자리 — 정의겸이 직접 작성 예정]",
    "[여기에 두 번째 문단을 쓰세요 — 원하는 만큼 문단을 추가할 수 있어요.]"
  ],

  signoff: "사랑을 담아,\n아들 정의겸 올림"  // 맺음말 (고정)
};
```

- `script.js`는 `LETTER.body` 배열을 순회해 문단 `<p>`를 생성하고 스크롤 등장 애니메이션을 붙입니다.
- 텍스트 파일 분리가 부담이면 대안: `index.html` 안에 편지 영역을 두고 명확한 HTML 주석으로 감싸기(아래 7번 마커 사용). 단 **content.js 방식(데이터 분리) 권장**.

## 7. 편지 본문 플레이스홀더 마커 (정확히 이대로 구현)

개발자는 편지 본문 자리를 **절대 임의로 채우지 말고**, 아래 마커를 그대로 넣어 정의겸이 쉽게 찾도록 합니다.

**본문 플레이스홀더 텍스트 (이 문자열을 그대로 사용):**

```
[편지 본문 자리 — 정의겸이 직접 작성 예정]
```

**HTML 안에 둘 경우 — 감싸는 주석도 함께:**

```html
<!-- ▼▼▼ 편지 본문 시작 — 정의겸이 직접 작성할 부분 ▼▼▼ -->
<!-- content.js 의 LETTER.body 배열을 수정하세요. 아래 문단은 예시 플레이스홀더입니다. -->
<p>[편지 본문 자리 — 정의겸이 직접 작성 예정]</p>
<!-- ▲▲▲ 편지 본문 끝 ▲▲▲ -->
```

- 플레이스홀더는 화면에서도 **눈에 띄게** 보여야 함(연한 배경/점선 테두리 등 개발 중 표시). 단, 실제 텍스트가 채워지면 자연스러운 본문 스타일로.
- 최종 편지 문구는 **정의겸 본인만** 작성. 개발자/테스터는 이 문구를 창작하지 않음.

## 9. 접근성 / 마무리 체크

- `prefers-reduced-motion: reduce` 대응(타이핑/슬라이드 생략, 즉시 표시).
- 색 대비 충분(본문/배경).
- 시맨틱 마크업(`<main>`, `<section>`, `<h1>`, `<p>`), `lang="ko"`.
- 폰트 로딩 실패해도 폴백 폰트로 읽힘.
- 가로 스크롤 없음, 모든 화면폭에서 깨지지 않음.
```
