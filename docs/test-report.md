# 테스트 리포트 — 엄마 생신 축하 편지 페이지

> 테스터 검수 문서. 대상 커밋: 구현본(index.html / style.css / script.js / content.js).
> 두 축으로 평가: **① 디자인 평가**, **② 기능 평가**.

---

## 0. 테스트 방법 (How)

- **실제 브라우저 렌더링 확인** (정적 리뷰만 하지 않음): 로컬 `file://` 경로의 `index.html`을
  데스크톱 브라우저(Microsoft Edge)에서 직접 열어 화면을 육안 확인했습니다.
  - 데스크톱 폭(~1450px)과 **모바일 폭(412px)** 두 가지에서 스크린샷으로 확인.
  - Intro 타이핑 완료 상태, 스크롤 힌트 화살표(↓), 스크롤에 따른 섹션 등장(fade+slide),
    본문 플레이스홀더 카드, 맺음말 줄바꿈, 한글 단어 단위 줄바꿈(`word-break: keep-all`),
    가로 스크롤 없음 등을 실물로 확인.
  - Google Fonts(Nanum Myeongjo)가 CDN에서 정상 로드되어 명조체로 렌더됨을 확인.
- **정적/구조 리뷰**: HTML/CSS/JS를 손으로 트레이스. id/class 상호참조, script 로드 순서,
  IntersectionObserver 로직(중복발화/unobserve), 타이핑 타이머/정리, 폴백 경로,
  색 대비(WCAG 계산)를 점검.
- **구문 검사**: `node --check content.js`, `node --check script.js` — 둘 다 통과.

> 참고: Chrome 자동화(MCP) 확장이 연결돼 있지 않아 콘솔 로그를 프로그램적으로 수집하지는
> 못했습니다. 대신 모든 JS 경로(타이핑, 콘텐츠 주입, 스크롤 등장)가 실제로 정상 동작하는 것을
> 화면으로 확인했고, 렌더 결과에 오류 흔적(빈 섹션/미주입 텍스트/레이아웃 붕괴)이 없었습니다.

## 1. 디자인 평가 (Design)

전반적으로 스펙의 의도("밤에 조용히 켜서 한 줄씩 읽는 손편지")를 잘 구현했습니다.

**좋은 점 (스펙 부합)**
- **무드/색상**: 배경 크림(#FDF6EC), 본문 다크 세피아(#4A3F35), 강조 브라운(#8C5A3C)로
  따뜻하고 정적인 손편지 분위기가 실제 화면에서 잘 살아 있음. 다크톤 없음(밝고 편안).
- **타이포그래피**: Nanum Myeongjo 명조가 CDN으로 정상 로드되어 편지다운 정중함이 느껴짐.
  폴백(`Apple SD Gothic Neo`, `Malgun Gothic`, serif)도 지정됨.
- **가독성**: 본문 `line-height: 1.9`, 자간 `0.02em`, 넉넉한 문단 여백(2.2rem) — 어머니가
  천천히 읽기 좋은 호흡.
- **모바일 우선**: `max-width: 640px` 컨테이너, 좌우 패딩, `word-break: keep-all`로 한글이
  단어 단위로 자연스럽게 줄바꿈됨(412px에서 "원하는 만큼 / 문단을 추가할 수 있어요." 확인).
  가로 스크롤 없음.
- **애니메이션 느낌**: Intro 타자기 효과(55~70ms, 약간의 랜덤으로 손맛), 커서 깜빡임 후
  조용히 사라짐, 스크롤 힌트 화살표 bounce, 각 섹션의 fade+slide-up 등장, 본문 문단 stagger(0.1s)
  — 요란하지 않고 잔잔함.
- **맺음말**: 우측 정렬 + `\n` 줄바꿈("사랑을 담아, / 아들 정의겸 올림")이 편지 서명처럼 보임.
- **플레이스홀더 가시성**: 본문 자리가 점선 테두리 + 살짝 진한 크림 배경 + 이탤릭 흐린 세피아로
  **한눈에 '아직 채울 자리'임이 분명**함. (스펙 7번 요구 충족)

**디자인상 참고/권장 (버그 아님, 정의겸·디자이너 판단 사항)**
- **Nanum Pen Script 폰트가 로드되지만 실제로 사용되지 않음**: `<head>`의 Google Fonts
  링크와 `--font-hand` 변수는 있으나 어디에도 적용돼 있지 않음. 손글씨 포인트를 쓸 계획이
  없다면 링크에서 `Nanum+Pen+Script`를 빼면 모바일 로딩이 조금 가벼워짐. 나중에 제목/맺음말
  포인트로 쓸 여지를 남기려면 그대로 둬도 무방. → **디자이너 판단에 맡김(미수정).**
- 플레이스홀더 카드 안의 이탤릭 흐린 세피아 텍스트는 대비가 다소 낮으나, 이는 "임시 자리"임을
  드러내려는 **의도된 개발용 표시**이며 실제 편지 텍스트로 채워지면 자연 본문 스타일로 바뀌므로
  문제 없음.

## 2. 기능 평가 (Functional)

**정상 동작 확인**
- **id/class 상호참조 일치**: `typed`, `cursor`, `scrollHint`, `birthdayText`, `signoffText`,
  `letterBody`가 HTML↔JS에서 모두 일치. `.reveal`/`.body__paragraph` 셀렉터도 CSS와 일치.
- **script 로드 순서**: `content.js`(데이터) → `script.js`(로직) 순서 정확. `LETTER`가 전역
  렉시컬 바인딩으로 script.js에서 정상 참조됨.
- **콘텐츠 주입**: 생신 축하/맺음말 textContent 주입, 본문 문단 `LETTER.body` 순회 생성 정상.
- **IntersectionObserver**: `threshold: 0.15`, 진입 시 `is-visible` 부여 후 `unobserve`로
  **중복 발화 없음 / 한 번 등장 후 유지**. Intro가 100vh라 birthday/이후 섹션은 로드 시
  뷰포트 밖 → 스크롤 시 자연 등장. 실물에서 확인됨.
- **타이핑 타이머**: 인덱스 순회하며 `textContent` 누적, 완료 후 1200ms 뒤 커서 숨김.
- **폴백 경로**:
  - `IntersectionObserver` 미지원 시 전 대상 즉시 `is-visible`.
  - `prefers-reduced-motion: reduce` 시 타이핑 생략·전체 텍스트 즉시 표시, 스크롤 등장도
    CSS/JS 양쪽에서 즉시 표시.
- **스크롤 힌트**: `scrollY > 0` 최초 감지 시 fade-out + 리스너 제거(1회성). passive 리스너.
- **접근성**: `lang="ko"`, 시맨틱(`<main>/<section>/<h1>/<p>`), 커서·화살표 `aria-hidden`.
  본문/배경 대비 약 9.6:1(AAA), 강조 브라운/배경 약 5.2:1(AA 통과) — 계산으로 충분.
- **GitHub Pages 배포성**: 상대경로(`style.css`, `content.js`, `script.js`)만 사용, 절대
  로컬 경로·서버·빌드 의존 없음. 정적 서빙 그대로 동작. (폰트만 외부 CDN — 오프라인 시 폴백)

**발견한 결함 → 수정함**
- 🐞 **[수정 완료] `prefers-reduced-motion` 시 커서 `|`가 영구히 남던 문제.**
  - 증상: 모션 최소화 사용자에게는 타이핑을 건너뛰고 전체 텍스트를 즉시 표시하는데, 이때
    커서를 숨기는 로직이 없어 "엄마," 뒤에 **깜빡이지 않는 정적 `|` 막대가 계속 남음**(오타처럼 보임).
  - 원인: `runTyping()`의 reduced-motion 분기에서 텍스트만 채우고 `return` — 정상 경로에만
    있는 `cursorEl.classList.add("is-hidden")`가 이 분기엔 없었음.
  - 수정: `script.js`의 해당 분기에 `if (cursorEl) cursorEl.classList.add("is-hidden");` 추가.
    기존에 정상 동작이 검증된 `is-hidden` 메커니즘을 재사용한 1줄 타깃 수정. `node --check` 통과.

## 3. 남은 항목 (정의겸 / 사용자 결정 필요, 미수정)

- **편지 본문 텍스트(플레이스홀더)**: `content.js`의 `LETTER.body` 배열은 의도적으로
  플레이스홀더(`[편지 본문 자리 — 정의겸이 직접 작성 예정]` 등)로 남겨져 있음. 이는 스펙 7번
  규칙대로 **정의겸 본인만 작성**해야 하는 부분 → 테스터가 채우지 않음. (버그 아님)
  화면상 점선 카드로 '아직 채울 자리'임이 분명히 드러나도록 되어 있음을 확인함.
- **Nanum Pen Script 폰트 로드 여부**: 위 1번의 권장 참조. 손글씨 포인트 사용 계획에 따라
  디자이너가 유지/제거 결정. (현재 미사용이지만 오류는 아님 → 미수정)

## 4. 결론

- **디자인**: 스펙의 따뜻하고 진심 어린 손편지 무드를 잘 구현. 데스크톱·모바일 모두 깨짐 없음.
- **기능**: 구조/로직 정상, 폴백 정상, 배포 가능. 발견된 실제 결함 1건(reduced-motion 커서)은
  수정 완료.
- 남은 것은 정의겸의 **편지 본문 작성**뿐. 그대로 `content.js`의 `LETTER.body`만 고치면 됨.

# 라운드 2 — Deluxe Edition 재검수 (밤하늘/글래스/컨페티)

> 대상 커밋 598474c(구현) / 6c5d434(스펙 업그레이드). 컨셉 변경: 딥네이비→플럼 그라디언트 +
> CSS 별빛 + 글래스모피즘 카드 + AOS 스크롤 등장 + 서명부 canvas-confetti 피날레.

## R2-0. 테스트 방법 (How)

- **실제 브라우저 렌더링(Edge, file://)** 로 데스크톱(~1320px 창)과 모바일(창폭 ≈ CSS 417px)
  두 폭에서 스크린샷 확인. Intro(그라디언트+별빛+타이핑), 스크롤 후 글래스 카드/골드 글로우/
  플레이스홀더/서명, **서명부 진입 시 컨페티 발사**를 실물로 관찰.
- **정적/구조 리뷰 + 라이브러리 소스 검증**: AOS 2.3.4 `aos.js`를 CDN에서 받아
  `disable()`(내부 `M()`) 동작을 직접 확인 — reduced-motion 분기의 안전성 실증(아래 R2-2).
- **구문 검사**: `node --check content.js/script.js` 통과.
- 한계: Chrome MCP 미연결로 콘솔 로그 프로그램 수집은 불가. 대신 DOM 접근성 트리로 전 콘텐츠
  주입 확인 + 모든 이펙트가 화면에서 정상 동작함을 확인(오류 흔적 없음).

## R2-1. 디자인 평가 (Design)

스펙의 "화려하지만 깔끔" 의도를 잘 구현. **배경은 화려, 편지 텍스트는 유리 위에 정적·선명**.

- **배경**: 딥네이비→플럼 그라디언트(45s 이동) + 순수 CSS 별빛 2레이어(흰/골드 트윙클)가
  실물에서 은은하게 반짝임. tsParticles 없이 CSS 대안 채택 — 스펙 3-1 B안 허용 범위, 저사양
  안전. 라이브러리 0개로 배경 처리한 판단 적절.
- **글래스 카드**: `backdrop-filter: blur(14px)` 실제 적용 확인(카드 뒤 별빛이 흐려짐).
  둥근 모서리·테두리·그림자로 유리 질감. 카드 위 아이보리/골드 텍스트 **또렷하게 읽힘**.
- **강조/글로우**: 생신 축하 문구 소프트 골드 + 은은한 text-shadow 글로우(등장 시 2회 펄스 후
  정지 — 과하지 않음). 서명도 골드 글로우.
- **피날레**: 서명 섹션 진입 시 canvas-confetti가 골드/크림/로즈로 좌우 2발 발사되는 것 실물
  확인. 절제된 양으로 촌스럽지 않음.
- **모바일**: 글래스 카드가 화면폭에 맞게 축소, `word-break: keep-all`로 한글 단어 단위
  줄바꿈("원하는 만큼 문단을 추가할 수 / 있어요."), 가로 스크롤 없음.
- **플레이스홀더**: **골드 점선 테두리 + 이탤릭**으로 화려한 비주얼 위에서도 '채워야 할 자리'가
  명확. 스펙 7번(이펙트로 가려지지 않게) 충족.

디자인상 참고(버그 아님): 플레이스홀더 내부 라벤더 이탤릭은 대비가 다소 낮으나 의도된 임시
표시(실제 편지 채우면 아이보리 본문으로 전환). Nanum Pen Script는 여전히 로드되나 미사용(선택).

## R2-2. 기능 평가 (Functional)

- **스크롤 등장(AOS + 이중 폴백)**: AOS 정상 초기화·등장 확인. `once:true`로 1회 유지.
  AOS JS 미로드 시 `[data-aos]`에 IntersectionObserver로 `aos-animate` 직접 부여하는 폴백
  경로 코드 정확(중복 발동 없음 — `unobserve`). 본문 문단은 fillStaticContent에서 먼저 생성 후
  init 순서라 AOS/IO가 정상 관찰.
- **컨페티 1회**: 서명 섹션 threshold 0.4 최초 진입 시 발사 후 `unobserve` → **재발사 없음**.
  `window.confetti` 없으면 조용히 return, `IntersectionObserver` 없으면 return, reduced-motion
  이면 return(+ `disableForReducedMotion:true`) — 라이브러리 실패해도 사이트 정상.
- **저전력 분기**: `max-width:767px` 또는 `hardwareConcurrency<=4`면 컨페티 120→70. 코드 정확,
  모바일에서 분기됨.
- **글래스 폴백**: `@supports not (backdrop-filter…)`에서 불투명 배경(rgba(30,20,50,.78))으로
  가독성 확보 — 코드 정확(미지원 브라우저에서도 본문 대비 유지).
- **prefers-reduced-motion(하이라이트 검증)**: 그라디언트/별빛/floatUp/커서/화살표/글로우 모두
  `animation:none`, `[data-aos]` 즉시 표시. **잠재 위험 실증 검증**: AOS.css의
  `[data-aos^='fade'][data-aos^='fade']{opacity:0}`(specificity 0,2,0)가 style.css의
  reduced-motion 오버라이드(0,1,0)보다 높아, AOS가 비활성일 때 콘텐츠가 숨겨질 우려가 있었음.
  → AOS 2.3.4 소스 확인 결과 `init`에서 `disable`이 참이면 `M()`이 호출되어 각 요소의
  `data-aos` 속성을 **제거**(removeAttribute)함 → 셀렉터 자체가 매칭 안 됨 → 콘텐츠 정상 표시.
  **숨김 버그 없음(실증 완료)**. (AOS 미로드 폴백 경로에서도 `aos-animate` 즉시 부여로 안전.)
- **CDN 실패 내성**: 폰트(폴백 serif), AOS(폴백 IO/즉시표시), confetti(생략) 모두 graceful
  degradation. body 배경 폴백 단색(#0F0C29)으로 배경 레이어 로드 전에도 깨지지 않음.
- **배포성/구문**: 상대경로 유지, CDN은 https 절대경로(정상), `node --check` 통과.
